# Task Navigator

依頼内容に適したスキルの連鎖（エントリスキル→チェックポイントでの後続スキル）を計画してから実行する手動エントリポイント。引数なしならスキルカタログを表示する。

## 既存設定との関係

- **Phase 0-5（@context/workflow-rules.md）**: 補完。複雑タスクではPhase 0-5が骨格であり、本スキルは「各Phaseでどのスキルを発火するか」のナビゲーションのみ担う
- **モデル自動発火**: 本スキルはuser-invoked専用。通常の依頼では従来通りモデル判断で個別スキルが発火する（本スキルは自動選択を置き換えない）

## ワークフロー

### Step 0: カタログの動的生成（両モード共通・毎回実行）

スキル一覧をハードコードせず、実行時にfrontmatterから取得する（スキルの追加・変更に自動追従させるため。真実源は各スキルのdescription）。**frontmatter先頭ブロックのみを対象にする**（単純なgrepは本文コードフェンス内の`description:`例を誤って拾い、`description: |`の複数行形式を取りこぼすため）:

```bash
for dir in ~/.claude/skills ./.claude/skills; do
  [ -d "$dir" ] || continue
  for f in "$dir"/*/SKILL.md; do
    echo "== $(basename "$(dirname "$f")")"
    awk '/^---$/{c++; next} c>=2{exit} c==1' "$f" \
      | awk '/^description:/{d=1} d && /^[a-zA-Z_-]+:/ && !/^description:/{exit} d{print}'
  done
done
```

加えて、システムプロンプトのAvailable skills（ビルトイン: code-review/verify/simplify等）も候補に含める。

- 完了基準: user-level・project-level・ビルトインの3ソースを確認済み

### Step 1a: カタログモード（引数なし）

スキルを用途カテゴリ別（例: 要件・設計 / 実装・大規模タスク / レビュー・品質 / コミット・PR / 指示ファイル管理 / その他）に整理し、各スキル1行（名前+いつ使うか）で表示して終了する。

- 完了基準: Step 0で取得した全スキルがいずれかのカテゴリに載っている

### Step 1b: ルーティングモード（引数あり）

1. **タスク分類**: 複雑タスク（複数ファイル・調査+実装・長時間）ならPhase 0-5を適用。単純タスクなら骨格なしで直行
2. **スキル連鎖の計画**: エントリスキルと、チェックポイント（計画完了時・実装完了時・コミット前・PR前等）で発火する後続スキルを1行ずつ列挙する。典型例:

```
例:「◯◯機能を作りたい」（複雑）
  /design-feature（要件定義）→ Phase 0-5開始（5+ファイルなら/large-task）
  → 実装 → /self-review → /commit → /create-draft-pr
例:「このPRを見てほしい」（単純）
  /pr-review のみ（チェーン不要）
```

3. **計画の提示**: チェーンをユーザーに1度提示する（数行。承認待ちはせず、明らかな解釈違いを訂正できる機会として）。該当スキルが無いステップは通常のツールで実施する旨を明記
4. **実行**: 計画順に各スキルを発動する（Claude Code: Skill ツール、Codex: `$skill-name`）。各スキル完了時にチェーンの現在地を確認し、次のステップへ進む。複雑タスクではチェーンをタスク管理機構（Claude Code: TaskCreate、Codex: plan）に登録し、05_log.mdに進行を記録する

- 完了基準: 計画したチェーンの全ステップが「実行済み」または「スキップ（理由を05_log.mdに記録）」になっている

## Gotchas

- **over-orchestration禁止**: 単純タスク（1ファイル修正・単発質問等）にチェーンを組まない。エントリスキル1つ、またはスキルなしの直接実行が正解のことが多い（チェーンを作ること自体が目的ではない）
- **スキル内部の手順が優先**: 発火した各スキルのワークフロー・完了基準はそのスキルに従う。本スキルは順序の管理のみ行い、各スキルの手順に介入しない
- **長時間タスクでのcompaction**: 本スキル本文は圧縮で失われ得るため、チェーンの状態は必ずタスク管理機構（Claude Code: TaskCreate、Codex: plan）/05_log.md側に持たせる（compaction後はそちらから復元する）
- **カタログの鮮度**: スキル一覧・境界の真実源は各スキルのdescription。本ファイルに個別スキル名の一覧表を追記しない（陳腐化するため。典型例の2つは説明用に留める）
