# メール（対外ビジネス）

協賛打診・取引先・会場/業者・イベント参加者など、社外宛の日本語メールを ukwhatn 本人の文体で書くためのリファレンス。

**出典**: 本人の送信メール全数分析（2022-01〜2026-07、約2,900通走査・地の文約400通精読、nxtendアカウント）＋本人FB（2026-07-16）。習慣は年々変化しており、**本人FB ＞ 直近実態 ＞ 過去実態** の優先で規範化している。

**注意**: SKILL.md STEP 2 の難語回避原則（「〜に関しまして」→「〜について」等）より**本リファレンスの定型が優先**。対外メールでは「〜につきまして」「〜させていただく」等の重めの敬語運用が本人の地の文そのもの。

## 骨格テンプレ

### 新規メッセージ

```
<正式社名（法人格込み）>
<名字>様

お世話になっております、特定非営利活動法人NxTEND 理事の渡邉でございます。

<（お礼／詫び、必要な場合のみ短く）>

<用件宣言: 「〜につきまして、ご相談させていただきたく、ご連絡いたしました。」>

<本文>

なにかご不明な点がございましたら、お気軽にお問い合わせくださいませ。

引き続き、何卒よろしくお願いいたします。
```

※所属・肩書は送信元アカウントに合わせる（上は NxTEND relation@ の例）。担当者不明なら宛名は「採用ご担当者様」「ご担当者様」。

### リプライ

```
<名字>様

お世話になっております、渡邉でございます。

<以下同じ>
```

- **リプライの宛名は「名字様」のみ**（社名・改行・役職なし）
- **名乗りは読点でつなぐ1文**。「お世話になっております。」で文を切らない
- 真の初対面は「突然のご連絡をお許しください。」＋名乗りは「〜にて理事を務めております、渡邉と申します。」＋紹介経路・面識を直後に明示
- 担当引き継ぎ時は名乗り直後に「〇〇に代わり、本件私が対応させていただきます。」

## 本文の書き方

- **1文1行**。長い文は読点・意味の切れ目で折り返して2行に。段落間は空行1、話題転換で空行2
- 話題転換「さて、」、補足「なお、」、追加「また、」「併せて」、依頼への接続「つきましては、」
- 主題は「〜につきまして（は）」で立てる。接続は連用中止「〜まして、」「〜ため、」「〜ので、」。**理由→依頼の順**
- 詳細情報は「---」区切り線で囲んだブロックに隔離: 「■ 見出し」＋「・」箇条書き＋全角スペースインデント＋「※」注記
- 複数質問への回答は相手の文を「> 」で引用再掲→直下に回答→「以上、回答とさせていただきます。」
- 締めに入る合図として「以上、」を独立で置く

## お礼・詫び

- お礼は**直前のアクションへの短い名詞句＋「ありがとうございます」**: 「早速のご確認、誠にありがとうございます。」
- **実績列挙を伴う独立お礼段落は書かない**。過去実績への言及は事実の流れに織り込む: 「2025年・2026年に「Matsuriba MAX」にご協賛いただいておりましたが、〜」
- 返信遅延には必ず冒頭で詫びる: 「ご返信が遅くなりまして、大変申し訳ございません。」連投は「五月雨のご連絡、大変申し訳ありません。」
- 詫びの2段階: 軽微=「申し訳ありません」／重め・自分の過失=「大変申し訳ございません」＋原因・対処を率直に
- 「賜り」は最上級の場面（新規協賛打診・年間協賛先）限定。「誠に」は広く使う。相手の詫びには「こちらこそ」

## 依頼・案内

- 主力: 「〜いただけますと幸いです。」（1通に複数回可）。強めの手続き指示は「〜いただきますようお願いいたします」
- クッション語をほぼ必ずセット: 「お手数ですが」「ご多忙の折大変恐縮ですが」「恐れ入りますが」
- 押しは弱く相手に裁量を渡す: 「お気軽にお申し付けください」。期限も「間に合わない場合はご相談ください」と開放。督促は理由付き
- 日程調整4点セット: 候補列挙＋「ご都合の良い日時を複数」＋所要時間明示＋「合わなければ別途調整」

## 締め

- 最頻: 「引き続き、何卒よろしくお願いいたします。」（読点入り）
- 新規・重い依頼は格上げ: 「ご多忙の折、大変恐縮ではございますが、何卒よろしくお願い申し上げます。」
- 対面直前: 「当日は何卒よろしくお願いいたします。」受領のみ: 「取り急ぎ、受領のご報告のみにて失礼いたします。」

## 語彙・記号・表記

- 【】=件名タグ・ブロック見出し、「」=イベント名・固有名詞、URLは単独行に素貼り
- 特徴語彙: 「委細承知いたしました」「ご放念いただけますと幸いです」「拝受いたしました」「ご教示」「ご査収」
- 開き: 「ください」かな固定（「下さい」不使用）。「いたします/致します」は本人も揺れているが**生成時はかな（いたします）に統一**
- **文末括弧での規模・補足表記は禁止**: ×「〜開催します（学生20〜40名規模）。」→ ○「学生20〜40名規模で福岡市内での開催を予定しており、」（数値は文中に織り込む）。括弧は「（cc: 〜様）」「（税抜）」等の但し書き専用

## 敬語レベル

- 基調は です・ます＋「させていただく」高密度。**「でございます」は名乗りのコピュラ専用**（本文では使わない）
- 「ございます／ございません」は存在動詞として多用（「問題ございません」「相違ございませんでしょうか」）
- **相手がカジュアルでも敬体を崩さない**。相手類型で変えるのは情報量と構造であって敬語レベルではない（業者宛は短文実務的、参加者宛はやや温かめ、でも型は同じ）
- 内部ML・チーム宛のみ簡潔（名乗りなし・指示1行）

## 場面別の型

- **辞退を受けたら**: 「承知いたしました」→検討への謝意→「また次回以降開催の際にお声がけさせていただければと存じます。」（次回への扉）
- **新規協賛打診**: フル名乗り＋紹介経路＋法人紹介＋資料＋オンライン説明の打診
- **受諾・確認**: 「〜とのこと、承知いたしました。」「ご認識に相違ございません」

## AIが本人流から外れやすいポイント（生成後に必ずチェック）

1. リプライでも社名フル宛名を書いてしまう → 名字様のみ
2. 名乗りを「お世話になっております。」で切って2文にしてしまう → 読点1文
3. お礼独立文＋実績列挙の営業構文を作ってしまう → 事実の流れに織り込む
4. 「継続接点: 〜」等のコロン付き構造化列挙・「延べ1,210名」等の数値アピール営業文体 → 使わない
5. 文末括弧の規模表記 → 文中に織り込む
