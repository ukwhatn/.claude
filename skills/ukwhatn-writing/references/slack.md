# Slack の書き分け

Slackは相手と用途で文体が最も大きく変わる。**雑談（砕け）**・**対外・公的・目上（完全敬語）**・**業務の依頼・確認・報告（端的）**の3レジスタを使い分ける。同じ同僚相手でも、雑談ならA、業務の依頼・確認ならCになる（相手だけでなく用途で選ぶ）。

---

## レジスタ A: 雑談（自分のtimes / 同僚との雑談）（砕け）

タメ口・常体＋終助詞。句点をほぼ打たず、改行や全角スペース `　` で区切る。1メッセージ＝1〜2行で連投。

### 特徴
- 文末: `〜な` `〜わ` `〜よ` `〜か`（`あったわ` `出るな` `戻るか`）
- ぼかし: `〜かも` `〜と思う` `〜気がする`
- `（）` で自嘲・オチ・本音補足
- 三点リーダ引き伸ばし `......` `‥…………` で脱力
- カスタム絵文字、`:emoji: < セリフ` 吹き出しネタ
- フィラー `あと、` `たぶん` `一応` `なんか` `べつに` `たしかに`

### 実例（原文ママ）
- `ご飯食べる暇がない〜`
- `なんか知らんうちにClaude Codeがいい感じになってるな　Zedから戻るか`
- `worktree、claudeにいじってもらうことしかしなくなった`
- `大変なんすよ……（）`
- `たしかに..........カスタム性というか載ってるOSかな........ならNASか.................`
- `べつにClaudeもCodexも契約したら良いんじゃないですか（）`
- `:ukwhatn_oni: < チェックインしてない席を使っている悪い新卒はいねが〜`

### 後輩へのコーチング（砕けつつ、答えでなく問いを投げる）
```
現状の深堀り自体はだいぶできててあんまり言うこと無いな（）

あと、「積極的にリードしない」と「介入せずに放置する」はイコールではないと思っていて、軌道修正はするけど意思決定はしない立ち回りの形を模索できるといいかもしれない（例えば ズレてると思ったら、答えは出さず「いまここズレてない？」って問いだけ投げる みたいな）
```
→ 指示でなく提案（`〜してみるのがおすすめです` `〜できるといいかもしれない`）。断定しない。

## レジスタ B: 対外・公的・目上（社外/全社/事務局/上長）（完全敬語）

ビジネス敬語。**「お疲れ様です。」で開始 → 文脈1〜2文 → 「〜でしょうか。」で問う → 「よろしくお願いいたします」で締め**。依頼・お詫びには `:bow-nya:` `:pray-nya:` `:man-bowing:` を1個クッションとして添える。

### 構成: 文脈 → 問い（結論先行ではなく、背景を置いてから用件）

### 質問・確認の定型
- 「〜でしょうか。」「〜という認識／理解で良いでしょうか。」「〜してもよろしいでしょうか。」
- 許可は必ず可否を問う形にする。自分の理解を確認して合意を取る。

### 実例（原文ママ）
```
<@kazuki.iwasaki> san
お疲れ様です。
Claude全社統一環境において、Claude DesktopからClaude Codeを利用する場合、TUIでは利用できるauto modeが利用できないようなのですが、こちらは意図した制限でしょうか。
```
```
お疲れ様です、DeNA Account Serviceのサービスコードを対象として参加させていただきたいのですが、一点（念の為）確認させてください。
```
- 依頼＋配慮: `こちら、何か操作方法が間違っていればご教示いただきたく、お手隙でご確認いただけますと幸いです :man-bowing:`
- 依頼の締め: `お手数おかけして申し訳ありません、よろしくお願いいたします :bow-nya:`
- お礼: `承知いたしました、ありがとうございます :pray-nya:` / `素早いご対応ありがとうございました！！`

### 配慮表現（的確に、ただし要件は曖昧にしない）
「お手数ですが」「お手隙で結構です」「念の為」「（メンション失礼します）」を使う。配慮しつつ、可否・対象・期限は具体で書く。根拠リンク（社内Wiki/Notion/Slackパーマリンク/GitHub）を併記する。

### 込み入った報告は構造化
背景を1文置いてから、箇条書き（`•`）と `=>` で「期待値 vs 実測」を対比する。
```
一応事象の詳細
• プロジェクトコード一覧で〜 => 出る
• 週次入力画面〜 => 出ない
```

## レジスタ C: 業務の依頼・確認・報告（自org部下/対等な実務相手）（端的）

立場が上・気心知れた実務相手への業務連絡には、敬語を薄くして端的に詰める。絵文字少なめ。同僚相手でも雑談はA・業務連絡はCと用途で分ける。

### 実例（原文ママ）
- 催促: `本件何らかアップデートありますか？`
- 引き継ぎ依頼（砕け）: `九州のMTGマジで1回も出たことないので雑に引き継ぎをいただきたく.........................`
- 判断の共有（口語）: `既にご退職されているようですが、伊藤さんから繋げてもらうのが丸いという結論でした`
- 回答・注意喚起: `postman、アカウント作ると強制的にエンドポイントやmessage bodyの情報がクラウド同期されるので、そこだけ注意必要だと思います`
- 前提の問い返し: `「geminiを利用できる状態」=「GeminiシリーズのAPIを叩ける状態」ですか？`

---

## 共通の注意

- **「業務だから一律丁寧」ではない**。相手と立場で B（社外・目上）と C（部下・対等）を切り替える。
- 質問への回答では、まず相手の前提を `=` で言い換え確認してから答えることがある（認識ズレ防止）。
- 長い三点リーダ `.....` はレジスタBでもトーンダウンに稀に出る。
- 締めの定型（よろしくお願いいたします）はレジスタBのみ。内輪では締めない or 軽く。
- `（）`・三点リーダ・全角スペース・フィラーは「らしさ」を出すが、**読みやすさを落とすので使いすぎない**（1メッセージに1〜2個まで。レジスタA中心、B/Cでは原則使わない）。
