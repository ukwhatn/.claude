## 統合レポート（80_review.md）

### 初回レビューの出力形式

```markdown
# ドキュメントレビュー統合レポート（第N回）

## レビュー対象
- 対象: <ファイルパス>
- 参照: <参照ファイルパス>
- 前回: <前回80_review.mdパス or なし>

## レビュー結果サマリー

| Severity | 件数 |
|----------|------|
| Action Required | X件 |
| Recommended | X件 |
| Minor | X件 |

## Action Required（X件）
### AR-1: <タイトル>
- **検出Agent**: <Agent名>（X/6が検出）
- **設計箇所**: <セクション番号・行番号>
- **問題**: <詳細>
- **同種パターンの全箇所**: <他に同じ問題がある箇所のリスト>
- **修正案**: <具体的な修正内容>

## Recommended（X件）
[各指摘を記載]

## Minor（X件）
[各指摘を記載]
```

### 継続レビューの出力形式（--prev指定時）

初回の形式に加えて、以下のセクションを冒頭に追加する:

```markdown
## 前回レビューからのフォローアップ

### 前回AR修正状況
| 前回AR | 概要 | 修正状況 | 副作用 |
|--------|------|---------|--------|
| AR-1 | <概要> | 修正済み / 未修正 | なし / <副作用の内容> |

### 前回R/M未対応の再評価
| 前回指摘 | 概要 | 今回の判定 | 理由 |
|---------|------|-----------|------|
| R-1 | <概要> | 据え置き / AR昇格 / 修正済み | <理由> |

### AR推移
| ラウンド | AR | R | M |
|---------|-----|-----|-----|
| 第N-1回 | X | X | X |
| 第N回 | X | X | X |
```

## レビュートラッキング（81_review_tracking.md）

全ラウンドの指摘を横断的にトラッキングする。

```markdown
# レビュートラッキング

## 全指摘の横断トラッキング

### <指摘テーマ名>
| 項目 | 内容 |
|------|------|
| 初出 | R<N>-<AR/R/M>-<番号> |
| 内容 | <指摘内容> |
| 修正 | R<N>→R<N+1>間で修正済み |
| 再燃 | R<M>-<AR/R/M>-<番号>: <関連する新指摘> / なし |
| 最終状態 | 解決済み / 未対応(R) / 未対応(M) |
| 意思決定 | <ユーザー判断があれば記載> |
```

更新ルール:
- 新しい指摘 → 新規エントリ追加
- 既存指摘の再指摘 → 「再燃」フィールドに追記
- 修正が確認された → 「修正」フィールドに記録
- ユーザーが判断した → 「意思決定」フィールドに記録

## 収束状況の報告

継続レビューの収束判定（Step 9）でユーザーに提示する。

```markdown
## 収束状況

| 指標 | 値 | 判定 |
|------|-----|------|
| 今回AR | X件 | - |
| 前回AR残存 | X件 | 0なら✓ |
| R→AR昇格 | X件 | 0なら✓ |
| 修正副作用による新規AR | X件 | 0なら✓ |
| 同種パターン再指摘 | X件 | 0なら✓ |

### 収束条件
- [ ] Action Required = 0
- [ ] 前回R/Mの未対応分が新たなARに昇格していない
- [ ] 修正の副作用による新規ARが発生していない
```
